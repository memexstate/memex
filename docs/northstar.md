# Memex Northstar

> Memex is a continuity runtime for reasoning across time.

Memex preserves structured continuity state so reasoning does not restart every session.

Models perform reasoning compute.

Memex preserves the continuity structures that allow reasoning to continue.

---

# The Architectural Boundary

Memex is not a reasoning engine.

Memex provides:

- structured continuity state
- continuity across time
- memory structures
- operational grounding
- resumable execution state
- continuity regulation

Models provide:

- inference
- synthesis
- planning
- language reasoning
- interpretation

The reasoning engine may change.

The continuity structure must remain stable.

---

# Compass

Every Memex system begins with a Compass.

Compass defines what the system is trying to become.

Examples may include:

- research systems
- development systems
- reasoning systems
- operational systems
- assistant systems

Compass acts as the persistent orientation layer for continuity.

Without Compass:

- deviation has no meaning
- alignment becomes unstable
- long-horizon continuity drifts

All continuity regulation is evaluated relative to Compass.

---

# Snapshots

Snapshots are the atomic unit of continuity.

A snapshot captures structured system state at a moment in time.

```text
snapshot₀ → snapshot₁ → snapshot₂ → snapshot₃
```

The newest snapshot represents the active reasoning context.

Older snapshots move into continuity memory.

Snapshots preserve:

- working state
- continuity lineage
- operational grounding
- runtime state
- continuity carriers
- validated continuity structure

Memex treats snapshots as structured continuity cells rather than conversational history.

---

# The Snapshot Spine

Time in Memex is structured as a sequence of snapshots.

```text
present → past
```

The active snapshot represents the present moment.

Previous snapshots form continuity history.

This allows reasoning continuity to persist across sessions, runtime boundaries, and execution cycles.

---

# Trails

Memex preserves continuity through trails.

## Main Trail

The Main Trail preserves structured continuity history across snapshots.

This trail forms long-horizon continuity memory.

## Operational Trail

The Operational Trail preserves observed runtime reality.

Examples include:

- file changes
- command results
- runtime events
- artifact creation
- repository changes
- execution evidence

Observed operational reality always outranks interpretation.

The Operational Trail exists to keep continuity grounded in observable system behavior.

---

# Reality Grounding

Memex treats observed reality as a primary continuity constraint.

The system attempts to remain grounded in:

- observed runtime behavior
- filesystem state
- operational evidence
- execution ordering
- validated continuity state

The runtime intentionally avoids replacing observed reality with generated interpretation.

Reality must remain inspectable.

---

# Loops

Continuity is regulated through loops.

## Present Regulation

The runtime continuously compares:

- expectation
- observation
- continuity state
- operational reality

This allows continuity to remain aligned with observed conditions over time.

## Long-Horizon Improvement

Long-horizon processes analyze continuity across snapshots.

Examples may include:

- reflection
- retrieval
- compression
- pattern analysis
- continuity optimization

These processes influence future continuity state while preserving historical continuity integrity.

---

# Temporal Compression

Continuity memory changes resolution over time.

```text
moments → days → weeks → months → years
```

Recent continuity remains detailed.

Older continuity becomes increasingly compressed while preserving structural continuity shape.

The continuity structure remains stable even as time scale changes.

---

# Rehydration

Rehydration reconstructs active continuity state from stored snapshots.

The runtime treats rehydration as:

```text
deterministic continuity reconstruction
```

not:

```text
memory approximation
```

Missing continuity remains visible.

The runtime intentionally avoids:

- hidden inference
- fabricated continuity
- silent semantic rewriting
- invented historical state

The system prefers explicit gaps over fabricated continuity.

---

# Rollback

Snapshots act as continuity anchors.

If continuity integrity degrades, the system may restore a previous stable snapshot and resume from validated continuity state.

Rollback exists to preserve continuity stability during degraded runtime conditions.

```text

with stable semantics

regulatable,
and resumable across time?
```
